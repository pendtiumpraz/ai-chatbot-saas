Setiap Request → Verify Signature
Setiap kali user akses app, backend memverifikasi JWT dengan Public Key. Jika invalid → AKSES DITOLAK.
Keamanan

Mengapa 100% Tidak Bisa Dibajak?

🛡️

RSA-256 Asymmetric Signing

Private key hanya ada di License Manager. Backend hanya punya public key. Mustahil membuat signed token tanpa private key.

🚫

Database Injection = GAGAL

INSERT/UPDATE data license langsung ke DB? Tidak ada signed_payload → otomatis dianggap expired & status "tampered".

📡

No Offline Activation

Tanpa koneksi ke License Manager, aktivasi PASTI gagal. Tidak ada fallback offline yang bisa dieksploitasi.

🔍

Tamper Detection + Auto-Lock

Jika signed payload dimodifikasi, sistem mendeteksi, set status "tampered", dan log CRITICAL alert ke admin.

📦

Package Type dari JWT

Paket (Basic/Pro/Enterprise) dibaca dari JWT, bukan kolom DB. Edit DB → tetap "Basic".

⏰

Expiry dari JWT

Tanggal expired dibaca dari signed JWT. Ubah kolom expires_at di DB → tetap expired sesuai aslinya.

Advanced LLM Technique

Structured JSON & Parsing

Minta AI jangan merespons dengan teks naratif "Halo, berikut datanya...", tapi wajib merespons dalam format Pure JSON Object.

Response AI:
{ "score": 85, "risks": ["Data Leak", "XSS"], "action": "block" }
🧩

Mengapa Harus JSON?

JSON adalah bahasa yang dipahami oleh Backend & Frontend. Ketika AI mengeluarkan JSON, output tersebut langsung bisa diolah oleh code kita (`JSON.parse()`) tanpa perlu dibaca oleh manusia terlebih dahulu.

Ini adalah kunci untuk "Menyematkan Kepintaran" ke dalam komponen aplikasi, mengubah Generative AI menjadi Deterministic AI!
